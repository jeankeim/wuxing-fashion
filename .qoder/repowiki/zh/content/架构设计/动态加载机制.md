## 核心组件
- 应用主控（App）：负责初始化、路由协调、全局事件、动态视图加载与控制器生命周期管理
- 路由系统（Router）：拦截链接点击与浏览器前进后退，维护当前路由状态并派发路由变化事件
- 控制器基类（BaseController）：提供 mount/unmount 生命周期、事件绑定与全局状态订阅能力
- 视图模板（views/*.html）：按需加载的静态 HTML 片段，作为视图容器
- Service Worker（sw.js v4）：离线缓存与 Stale-While-Revalidate 策略，移除数据文件预缓存，提升加载性能与可用性
- 全局状态（store）：集中管理应用状态，驱动视图与控制器的响应式更新
- 错误处理（error-handler）：统一包装异步函数与网络请求，提供用户友好的错误提示
- 数据管理（data-manager）：处理用户数据的导出导入，支持动态数据文件加载

**更新** Service Worker v4 版本移除了数据文件预缓存，采用更灵活的动态加载策略

## 架构总览
动态加载的整体流程如下：
- 应用启动时，预加载首屏视图并注册对应控制器
- 路由变化时，App 动态加载目标视图 HTML，注册控制器并卸载当前控制器
- 视图切换通过 CSS 类隐藏/显示与滚动重置实现
- Service Worker v4 在安装阶段预缓存核心资源，移除数据文件预缓存，在 fetch 阶段采用缓存优先策略，/data/ 路径直接走网络

```mermaid
sequenceDiagram
participant U as "用户"
participant R as "Router(路由)"
participant A as "App(应用主控)"
participant V as "视图模板(.html)"
participant C as "控制器(Welcome/Entry)"
participant S as "Service Worker v4"
U->>R : 点击导航/浏览器前进后退
R->>R : 解析路径/更新历史
R-->>A : 派发 routechange 事件
A->>A : loadView(目标视图)
A->>V : fetch 视图 HTML
V-->>A : 返回 HTML 文本
A->>A : DOM 插入/标记已加载
A->>A : registerController(目标视图)
A->>C : mount()
A->>A : switchView(目标视图)
A->>A : 隐藏其他视图/滚动至顶部
Note over S : v4版本：数据文件直接网络请求
Note over S : v4版本：核心资源缓存优先
```

**更新** Service Worker v4 版本的关键变更：移除数据文件预缓存，/data/ 路径直接网络请求，核心资源继续缓存

## 详细组件分析

### App 类与动态视图加载
- 预加载与按需加载
  - 预加载：应用初始化时预加载首屏视图，减少首次进入延迟
  - 按需加载：路由变化时再加载目标视图，避免一次性加载全部视图
- 视图 HTML 异步加载
  - 使用 fetch 获取视图 HTML 文本
  - 通过临时容器解析并提取首个元素节点，再插入到应用容器
  - 使用 Set 记录已加载视图，避免重复加载
- 控制器动态注册
  - VIEW_CONFIG 将视图 ID 映射到控制器构造函数与 HTML 路径
  - registerController 根据视图 ID 创建控制器实例并缓存
- 视图切换
  - switchView 隐藏所有视图，显示目标视图并重置滚动位置
- 错误处理
  - loadView 内部捕获异常并记录日志，保证应用稳定性

```mermaid
flowchart TD
Start(["开始: 路由变化"]) --> LoadView["loadView(viewId)<br/>检查已加载/获取配置/异步 fetch"]
LoadView --> ParseHTML["解析 HTML 并插入 DOM"]
ParseHTML --> RegisterCtrl["registerController(viewId)<br/>创建并缓存控制器"]
RegisterCtrl --> MountCtrl["mount 当前控制器"]
MountCtrl --> SwitchView["switchView(viewId)<br/>隐藏其他/显示目标/重置滚动"]
SwitchView --> End(["结束"])
```

## 性能考量
- 资源缓存优化
  - Service Worker v4 预缓存核心资源，显著降低二次访问延迟
  - **移除数据文件预缓存**，采用动态加载策略，提高缓存效率与数据更新速度
  - fetch 拦截采用缓存优先与后台更新策略，兼顾速度与新鲜度
- 并发控制
  - loadView 使用 Set 避免重复加载；建议在高并发路由切换时增加队列或去抖
- 错误处理
  - withErrorHandler 与 initGlobalErrorHandler 保障异常可控，减少页面崩溃
- 视图切换
  - switchView 仅切换 CSS 类与滚动重置，避免复杂动画导致卡顿
- 数据加载
  - **动态数据加载**减少初始包体积，提升首屏性能
  - 数据文件直接网络请求，确保数据实时性

## 故障排查指南
- 视图未显示
  - 检查视图 HTML 是否成功加载与插入 DOM
  - 确认 switchView 是否正确移除其他视图的 hidden 类并显示目标视图
- 控制器未挂载
  - 确认 registerController 是否被调用且控制器构造函数正确
  - 检查 BaseController.mount 是否执行 bindEvents 与 onMount
- 路由无效
  - 检查 ROUTES 配置与 navigateTo 是否正确派发 routechange
- 网络错误
  - 使用 withErrorHandler 包装异步函数，查看控制台错误日志
  - 确认 Service Worker v4 是否正常激活与缓存命中
- **数据加载问题**
  - 检查 /data/ 路径的网络请求是否正常
  - 确认数据文件格式是否符合预期
  - 验证动态加载逻辑是否正确执行

**最新优化成果**：
- Service Worker v4 版本升级，移除数据文件预缓存，采用动态加载策略

- 增强基础路径检测逻辑，支持更灵活的部署环境
- 提升了应用的性能表现与数据更新灵活性

后续可在高并发路由切换场景引入队列/去抖与更细粒度的缓存策略，进一步优化性能与一致性。
